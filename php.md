# PHP 代码习惯

- 优先遵守项目现有框架约定，不混用不同风格的 service、controller、repository 写法。
- 业务逻辑不要堆在 controller；controller 只做参数、权限、调用和响应组织。
- 数组结构、返回字段和接口参数要保持 contract 稳定，不随意改名或改 shape。
- 对可能为空的输入、数组 key、外部响应字段做显式处理，避免隐式 notice 或宽松类型误判。
- 类型转换要写清楚意图，不依赖 PHP 隐式转换处理业务边界。
- SQL、缓存 key、队列 topic、配置项命名要和项目既有模式一致。
- 公共逻辑优先复用项目已有 helper 或 service，不在多个业务类里复制粘贴。
- 删除未使用的 use、变量、私有方法和过期注释。
