# Go 代码习惯

- 条件允许时，优先使用当前稳定的 Go 最新版本。
- Go 第三方库非特殊情况不使用依赖 cgo 的库。
- 优先使用项目已有 package 边界和 helper；公共逻辑放到项目已有公共目录，不在业务目录复制。
- 错误处理保持直接，不吞错误，不包装成无法定位的文案。
- 接口 contract 变更保守演进；旧接口要保留时新增变体，不直接覆盖旧行为。
- 涉及第三方库行为时，优先查看当前项目 vendored 或 `go.sum` 锁定版本的实现。
- 批量机械替换前先搜索统一模式，改完跑受影响 package 的定向测试或构建。
