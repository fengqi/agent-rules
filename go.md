# Go 代码习惯

- 优先使用项目已有 package 边界和 helper，不为了“更通用”新增抽象。
- 公共逻辑优先放到项目已有公共目录，不在业务目录里复制 helper。
- 错误处理保持直接，不吞错误，不把错误包装成无法定位的文案。
- 命名要表达业务含义，不使用临时、含糊或和实际用途不匹配的变量名。
- 删除不再使用的 struct 字段、方法、接口实现和测试数据。
- 条件允许时，优先使用当前稳定的 Go 最新版本。
- 接口 contract 变更要保守演进；旧接口需要保留时，新增变体，不直接覆盖旧行为。
- 测试优先覆盖真实分支和边界条件，不为了覆盖率写无意义断言。
- 涉及第三方库行为时，优先查看当前项目 vendored 或 go.sum 锁定版本的实现。
- Go 第三方库非特殊情况不使用依赖 cgo 的库。
- 批量机械替换前先搜索统一模式，改完跑受影响 package 的定向测试或构建。
